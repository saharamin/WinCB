koNlpy 설치 및 사용 관련 error

1. JDK developer version 설치되어 있어야 함
2. JPype 설치되어 있어야 함
3. KoNlpy 설치되어 있어야 함

https://wikidocs.net/22488 이 페이지와 KoNlpy 오리지널 페이지 보면서 설치하면 됨.
이 때 error 관리
1. python version, 컴퓨터 os version, JDK version, JPype version의 bit가 모두 맞아야 함
- python version 및 bit 확인은
>>> import platform
>>> platform.architecture()
('64bit', 'WindowsPE')
>>>   

2. JAVA_HOME 설정해 주어야 함(java environment 설정)
- 시스템 고급 설정에서 환경 변수 설정 시 - 시스템 변수 안 건드리고 사용자 변수에만 설정하면 됨
- Path 설정도 필요 없음. 오로지 사용자 변수에 JAVA_HOME만 설정하면 됨
- jdk 디렉토리 이름까지만 제대로 입력하면 그 밑에 bin/server같은 하위 디렉토리 이름은 굳이 입력 안 해도 됨(알아서 찾아감)
- 혹시 그래도 안 된다면 하위 변수를 정확히 입력해서 해 볼 필요는 한번쯤은 있음
- JAVA_HOME 설정 하고 컴퓨터를 rebooting해야 그 때 부터 올바로 작동됨
- Java 설치 완료 하기 전에 중간에 JAVA_HOME부터 설정하라는 이야기가 있는데 이것은 말이 안 되거나 rebooting과 관련된 이야기 같으며 rebooting하면 해결되는 이야기
- 이것이 제대로 되어 있지 않으면 No JVM shared library file (jvm.dll) found와 같은 error가 남

3. JAVA_HOME이 제대로 되어 있는데 error
- [WinError 126] JVM DLL not found: C:\Program Files\Java\jdk-15.0.2\bin\server\jvm.dll
- 위와 같이 jvm.dll이 있는 경로를 정확히 찾는데도 불구하고 file이 없다고 하는 경우
- JAVA_HOME은 제대로 된 것임. 그러나 jvm file 자체에 오류가 있음
- 이 문제는 jdk 15.0.2가 문제 였고 oracle 가입하고 11 버전으로 내려서 해결

4. JPype 관련
- 이렇게 했더니 JVM DLL not found error는 사라졌으나 jvm character가 어쩌고하는 error가 났는데 Jpype(java와 python 연결 역할)의 jvmfinder file에서 os에 설정된 JAVA_HOME 이름을 정상적으로 가져오지 못하는 것으로 판단되었음
- JPype을 conda로 설치했었는데(어떤 site 보고)
- 원래 위의 책에서 하라고 한대로 pip install로 맞는 버전 whl 찾아서 다시 JPype 설치함
- 그랬더니 모든 에러 해결

결론(주의점)
1. python version, 컴퓨터 os version, JDK version, JPype version의 bit가 모두 맞아야 함
2. JDK 특정 버전 안 됨. 
3. JPype은 conda로 깔지 말고 JPype 페이지에서 python과 맞는 버전, bit 찾아서 다운로드 후 pip install로 설치할 것
